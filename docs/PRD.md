# PRD — Гантелька

## Что это

Мобильный (PWA, max 480px) персональный тренировочный блокнот. Пользователь ведёт свои тренировки в зале: заводит упражнения, складывает из них тренировки, во время сессии фиксирует подходы (раз × кг).

## Зачем

Заменить заметки на телефоне для тех, кто системно тренируется/ Главная ценность — прогрессия нагрузки и история подходов на одном экране.

## Целевая аудитория

— Системный тренирующийся (не новичок «попробую месяц», и не профи с тренером).
— Любит порядок и удобный UX, не любит подписки.

## Состояния и флоу пользователя

### Новичок
Login → DevSelect → empty Home («нет тренировки?») → «начать пробную» → Workouts (4 пробные тренировки) → тап на тренировку → детальный просмотр → «запустить» → WorkoutSession → «завершить» → дата следующей → «УРА!» → Home (с тренировкой).

### Старичок (с тренировкой)
Login → DevSelect → Home («день ног ягодицы») → «начать тренировку» → детальный просмотр (только запустить + back) → «запустить» → WorkoutSession → …

### Сессия тренировки
Аккордеон упражнений (первое раскрыто). В раскрытом: подходы (раз × кг + trash), `(i)` инфо упражнения, `+ подход`, `готово` (зачёркивает имя). Sticky внизу — «завершить» → дата следующей → success.

## Реализовано

- Лендинг-логин (без реального OAuth)
- DevSelect (временный экран для тестирования — убрать перед запуском)
- Home (2 состояния: empty / with-workout)
- Workouts: список активных, архив, создание/редактирование с DnD, превью с «запустить»
- Exercises: список, фильтр (группа мышц + только мои), поиск, создание, детальный просмотр, добавление в тренировку
- WorkoutSession: подходы, info, готово, завершение с датой следующей тренировки, success-экран
- Единое хранилище через React Context + useReducer
- Пробные тренировки помечены `isTrial: true`, у них только «запустить»

## Сознательно НЕ сделано (пока)

- **Персистентность.** Всё в памяти. Перезагрузил — данные пропали. Это блокер №1 для запуска.
- **Реальный Google OAuth.** Кнопка «войти через Google» — заглушка.
- **Сводка/прогресс.** Раздел есть как заглушка, графиков нет.
- **Профиль.** Только временная dev-кнопка.
- **Сплит** (последовательность тренировок). Удалён — было сложно для MVP, вернём отдельно если понадобится.
- **Уведомления / напоминания / стрики.**
- **Шеринг / экспорт.**
- **Логирование сетов в историю.** Поля «раз»/«кг» вводятся, но в историю не пишутся.
- **Десктоп/планшет.** Только мобайл, max 480px.

## Тон и язык

- UI — строчными (Anonymous Pro mono для заголовков), эмоционально, не казённо: «ура!», «начать пробную», «убрать в архив».
- Минимум модальных окон и popup'ов — на маленьком экране они мешают.
- Кнопки крупные (50–64px), для одной руки.

## Метрики, которые нас интересуют

(когда будет аналитика)
- Активация: % пользователей, завершивших первую сессию (нажали «завершить»)
- Retention D7: % вернувшихся через 7 дней
- Среднее число сессий в неделю на активного юзера
- % создавших свою (не пробную) тренировку
