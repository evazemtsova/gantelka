# Документация

## Продукт и архитектура

- [PRD.md](./PRD.md) — что за продукт, для кого, что реализовано, что сознательно не сделано
- [TDR.md](./TDR.md) — технический обзор: стек, архитектура, структура файлов, модель данных

## Правила кода

- [PRINCIPLES.md](./PRINCIPLES.md) — универсальные принципы (общие для фронта и бэка)
- [FRONTEND.md](./FRONTEND.md) — правила фронта (React, TS, CSS, UI-компоненты)
- [BACKEND.md](./BACKEND.md) — правила бэка (Supabase + Vercel)
- [BACKEND_PLAN.md](./BACKEND_PLAN.md) — живой план миграции на бек, по фазам

## Журнал

- [CHANGELOG.md](./CHANGELOG.md) — журнал изменений (новые записи сверху)

## Кому что читать

| Я… | Читать |
|---|---|
| Новый разработчик в проекте | PRD → TDR → PRINCIPLES → FRONTEND/BACKEND (по специальности) |
| Делаю фронт-фичу | FRONTEND + соответствующий раздел TDR |
| Делаю бек-фичу | BACKEND + соответствующий раздел TDR |
| Удаляю/меняю архитектуру | PRINCIPLES § 8 + обновить TDR + CHANGELOG |
| Любое заметное изменение | Добавить запись в CHANGELOG |
| Продакт/дизайнер | PRD |
